# Security Policy

## Supported Versions

Orion is currently under active development. Security updates are applied to the latest version only.

| Version       | Supported |
| ------------- | --------- |
| Latest (main) | Yes       |
| Older builds  | No        |

Users are strongly encouraged to use the most recent version of Orion to ensure security and stability.

---

## Reporting a Vulnerability

If you discover a security vulnerability, do not open a public issue.

### How to report

Send a detailed report to:
**[wayvsoftware@gmail.com](mailto:wayvsoftware@gmail.com)**

Include:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Screenshots or proof-of-concept (if applicable)

---

## What to expect

* **Response time:** within 48 hours
* **Initial assessment:** within 3–5 days
* **Fix timeline:** depends on severity

You will be informed whether:

* The vulnerability is accepted and being worked on
* Additional information is required
* The report is declined (with explanation)

---

## Disclosure Policy

* Please do not publicly disclose vulnerabilities before a fix is released
* Once resolved, we may acknowledge your contribution

---

## Scope

This policy applies to:

* Core Orion codebase
* Official plugins and extensions
* Installer and distribution files

---

## Out of Scope

* Issues in third-party libraries (report to their maintainers)
* Non-security bugs (use GitHub Issues instead)

---

## Commitment

Security is taken seriously. All valid reports will be investigated and addressed to maintain a safe and reliable environment for users.
